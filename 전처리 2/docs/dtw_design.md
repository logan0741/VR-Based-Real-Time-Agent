# DTW 계산 설계 및 개선 과정

## 1. DTW란 무엇인가

DTW(Dynamic Time Warping)는 길이가 다른 두 시계열 시퀀스를 최적으로 정렬하여 유사도를 측정하는 알고리즘이다.

본 시스템에서는 사용자의 정규화된 관절 시퀀스와 전문가의 관절 시퀀스를 DTW로 비교하여 운동 품질 점수를 산출한다.

---

## 2. n_frames 도입 배경

### 문제: 매 프레임 전체 시퀀스로 DTW 계산

초기 구현에서는 매 프레임마다 지금까지 누적된 전체 사용자 시퀀스를 DTW에 입력했다.

```
프레임 30:  DTW(사용자 30프레임,  전문가 57프레임)
프레임 100: DTW(사용자 100프레임, 전문가 57프레임)
프레임 200: DTW(사용자 200프레임, 전문가 57프레임)
```

DTW의 연산 복잡도는 O(M × N)이므로, M(사용자 프레임 수)이 증가할수록 계산량이 선형으로 늘어났다.
그 결과 운동이 진행될수록 FPS가 점점 하락하는 문제가 발생했다.

### 해결: 슬라이딩 윈도우 n_frames 도입

실시간 점수는 "지금 이 순간 얼마나 잘하고 있나"를 나타내므로, 전체 과거를 볼 필요가 없다.
최근 n_frames개의 프레임만 사용하면 연산량을 O(n_frames × N)으로 고정할 수 있다.

```
프레임 30:  DTW(사용자 최근 30프레임, 전문가 57프레임)  ← 고정
프레임 100: DTW(사용자 최근 30프레임, 전문가 57프레임)  ← 고정
프레임 200: DTW(사용자 최근 30프레임, 전문가 57프레임)  ← 고정
```

## 3. 전문가 영상 프레임 수(57)를 n_frames으로 쓰지 않는 이유

### 이유 1: 운동 속도 차이

전문가와 사용자의 운동 속도가 다를 수 있다.

| 구분 | 스쿼트 1회 소요 프레임 |
|------|----------------------|
| 전문가 | 57프레임 (~2.4초) |
| 사용자 A | 40프레임 (빠름) |
| 사용자 B | 80프레임 (느림) |

DTW는 길이가 다른 두 시퀀스를 자동으로 정렬하는 알고리즘이므로, n_frames을 전문가 프레임 수와 일치시킬 필요가 없다. 사용자가 느리거나 빠르더라도 DTW가 최적 정렬 경로를 찾아 비교한다.

### 이유 2: 실시간 점수의 역할

실시간 점수는 절대적으로 정확한 평가가 아니라, 빠른 피드백을 위한 **대략적인 지표**다. 정확한 평가는 운동 1회가 완료된 시점에 별도로 수행한다(→ 4절).

### 이유 3: 성능

n_frames이 클수록 DTW 연산 시간이 증가한다.

| n_frames | DTW 시간 (측정값) |
|----------|-----------------|
| 30 | ~57 ms |
| 57 | ~108 ms (추정) |

목표 FPS(15)를 달성하려면 적절한 크기로 제한해야 한다.

## 4. 회차별 DTW 도입 배경

### 문제: 실시간 점수만으로는 정확한 평가 불가

슬라이딩 윈도우 실시간 점수는 동작의 일부만 보므로, 운동 1회 전체에 대한 정확한 평가가 어렵다.

| n_frames | 실시간 평균 점수 | 회차별 점수 | 차이 |
|----------|----------------|------------|------|
| 30 | [87, 86, 89, 88] | [95, 92, 95, 94] | 평균 6.5 |

실시간 점수는 회차별 점수보다 일관되게 낮다. 이는 윈도우가 동작 전체를 포착하지 못하기 때문이다.

### 해결: 운동 1회 완료 시점에 전체 구간 DTW 계산

RepDetector가 운동 1회 완료를 감지하면, 해당 구간 전체 프레임으로 DTW를 1회 계산한다.

```
rep 1 완료 (프레임 10~82):
  DTW(사용자 72프레임, 전문가 57프레임) → 회차별 점수
```

이 방식의 장점:
- 운동 1회 전체를 보므로 정확도가 높음
- rep 완료 시 1회만 계산하므로 연산 부담이 적음
- 사용자마다 다른 운동 속도를 자연스럽게 처리

---

## 5. n_frames 최적값 탐색 방법

회차별 점수를 기준(ground truth)으로 삼아 각 n_frames 후보의 실시간 점수 정확도를 비교한다.

**절차:**
1. 동일한 테스트 영상으로 여러 n_frames 후보를 실험
2. 각 rep 구간 동안의 실시간 점수 평균을 계산
3. 회차별 점수와의 차이를 측정
4. 허용 오차 범위 내에서 가장 작은 n_frames 선택

실시간 점수는 절대값 정확도보다 **상대적 변화 추이(점수 증감)**가 의미 있으므로, 허용 오차 기준은 엄격하게 잡지 않아도 된다.

## 6. DTW 계산 주기(dtw_interval) 도입

### 문제: DTW가 매 프레임 실행됨

슬라이딩 윈도우로 연산량을 고정했더라도, DTW 1회 실행 비용이 ~57 ms로 높아 목표 FPS 달성이 어려웠다.

```
[Timing] predict : 17.4 ms
[Timing] DTW     : 55.1 ms  ← 병목
→ Avg FPS: 14.8
```

### 해결: N프레임마다 1회 계산

실시간 점수는 매 프레임 갱신될 필요가 없다. N프레임마다 1회 계산하고, 나머지 프레임은 이전 점수를 유지한다.

```python
"dtw_interval": 3  # 3프레임마다 DTW 1회 계산
```

| dtw_interval | 점수 갱신 주기 (실제 FPS ~30 기준) | Avg FPS |
|-------------|----------------------------------|---------|
| 1 (매 프레임) | 초당 30회 | 14.8 |
| 3 | 초당 10회 | 29.6 |

dtw_interval은 종목별로 config.py에서 관리한다.
