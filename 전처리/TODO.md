## [전체 완성 후 조정 필요] 점수 보정

### 문제
전문가 영상의 다른 버전 영상으로 테스트했을 때 회차별 점수가 71~86점으로 예상보다 낮게 산출됨.
현재 파라미터(`max_distance=1.0`)가 실제 사용 환경에 맞게 조정되지 않은 상태.

### 결정해야 할 사항
- `max_distance` 기준값 재조정 — 정규화된 몸통 길이 단위에서 실제 허용 오차와 맞는 값 탐색
- 관절별 `weights` 조정 — 점수를 낮추는 특정 관절이 있는지 확인 후 가중치 재배분 검토
- 점수 변환 함수 형태 검토 — 현재 선형 변환 외에 비선형(예: 제곱) 변환 고려 가능

### 제공 시점 가이드
**전체 파이프라인(08단계) 완성 후**, 실제 카메라 입력 환경에서 점수를 확인하며 보정한다.

## [피드백 단계에서 구현] 방향 포함 차이 행렬 (diff_matrix)

### 배경
현재 DTWComparator는 사용자와 전문가 관절 간 유클리드 거리(`dist_matrix`)만 반환한다.
거리는 "얼마나 다른지"는 알 수 있지만 "어느 방향으로 다른지"는 알 수 없어,
"무릎을 더 내리세요", "무릎을 더 앞으로 내미세요" 같은 방향성 있는 피드백을 생성할 수 없다.

### 필요한 것
DTWComparator에서 부호 포함 차이값(`user - expert`)을 Y축(상하)·X축(전후) 별로 반환하는 기능 추가.
`diff_matrix` shape: `(M, K, 2)` — 마지막 차원이 `[dy, dx]`.

### 주의사항
X축 부호의 의미는 촬영 방향(`normalizer_type`)에 따라 달라진다.
스쿼트 기본값은 `side_left` (화면 기준 왼쪽을 바라봄).
부호 해석 규칙은 피드백 엔진 구현 시점에 결정한다.

### 구현 시점
08단계 완성 후 피드백 엔진 추가 시 DTWComparator를 확장한다. 기존 `dist_matrix` 흐름은 그대로 유지.
